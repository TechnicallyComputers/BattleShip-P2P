# Known Bugs & Resolved Issues

This directory documents significant bugs encountered during the port, their symptoms, root causes, and fixes. When fixing a new bug, add an entry here so future sessions can recognize the same class of issue.

| 2026-04-22 | [fighter_appearr_body_frozen](fighter_appearr_body_frozen_2026-04-22.md) | **RESOLVED** — Mario / Link Appear intros rendered bodies frozen in Wait stance while entry effects (pipe, beam, capsule) animated correctly. Regression from 3161c13's walker hardening: the new "raw form also parses → reject" rule triggered on one-step End false positives (halfswap-corrupted u32s whose bits [31:25] happen to equal 0), causing the walker to reject halfswap-corrupted streams as "ambiguous native." Parser read the raw bytes, hit End immediately, terminated every body joint's animation on the first parse call. Fix: replace binary ambiguity rule with length comparison — raw must walk ≥ 2 commands to count as genuinely native, otherwise un-halfswap wins |
| 2026-04-20 | [particle_banks_silent_empty](particle_banks_silent_empty_2026-04-20.md) | **RESOLVED** — Every in-game particle effect (hit sparks, fire-flower fire, explosion flame, Samus fire, Mario coins, fireball sparkles, stage hazards, title/attract particles) silently failed to render. The 2026-04-08 DMA-segfault fix papered over the crash by registering a dummy bank without calling `lbParticleSetupBankID`, so every `lbParticleMakeChildScriptID` bounds check (`script_id >= sLBParticleScriptBanksNum[id]`) failed against `scripts_num == 0` and returned NULL silently. Fix: new `port/bridge/particle_bank_bridge.cpp` loads all 9 banks from `particles/*` in ssb64.o2r, byte-swaps the headers (bytecode is left alone — `lbParticleReadFloatBigEnd`/`ReadUShort` are already LE-safe), calls `lbParticleSetupBankID`. Dedup via pristine-path cache + scripts_lo-keyed working-copy eviction. Supersedes `particle_bank_dma_2026-04-08.md` |
| 2026-04-20 | [color_image_to_zbuffer_draws](color_image_to_zbuffer_draws_2026-04-20.md) | **RESOLVED (TextureRectangle only)** — Off-screen fighter magnifier bubble rendered as an opaque black square with the fighter icon inside. SSB64 uses an N64 trick that redirects the color image to the Z buffer to build an alpha-masked stencil before the visible XLU draw. Fast3D doesn't emulate address redirection — all draws go to the primary FB — so the mask draw landed as a visible opaque rectangle. Fix adds the `color_image == z_buf_address` skip (already present in `GfxDpFillRectangle`) to `GfxDpTextureRectangle`. `GfxSpTri1` still needs the same guard if `mvOpeningRoom` transitions ever regress |

| 2026-04-20 | [sprite_32bpp_tmem_swizzle](sprite_32bpp_tmem_swizzle_2026-04-20.md) | **RESOLVED** — RGBA32 sprites (SMASH logo, fighter portraits, any 32bpp) rendered with scanline shear because the TMEM-line-swizzle fix from 2026-04-10 hardcoded the 4b/8b/16b swap granularity (XOR-4) and skipped 32bpp entirely. Real fix: for 32bpp, swap 8-byte halves of 16-byte groups (XOR-16 at DRAM level) because `G_IM_SIZ_32b_LOAD_BLOCK = G_IM_SIZ_32b` puts each texel in two TMEM words via bank split |
